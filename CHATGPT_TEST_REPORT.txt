1. Fixed production issues
- Utrzymano działający flow `landing -> topic -> slot -> form -> payment -> confirmation -> room`.
- Domknięto publiczne fallbacki tak, aby flow klienta nie pokazywał już technicznych komunikatów o konfiguracji, env ani sekretach.
- Płatność w publicznym UI pokazuje teraz bezpieczne komunikaty zamiast surowych summary z runtime config.
- Checkout z niską lub błędnie skonfigurowaną kwotą nie zdradza już szczegółów panelu specjalisty.
- Materiały przed rozmową i pobieranie nagrania zwracają przy błędach konfiguracji bezpieczne, użytkownikowe komunikaty.
- Panel admina przy braku sekretu ochrony zwraca kontrolowany komunikat zamiast instrukcji z nazwą zmiennej env.
- Uzupełniono brakujące elementy SEO / accessiblity: `canonical`, `twitter` metadata i bezpieczne `rel="noopener noreferrer"` dla linków zewnętrznych.
- Uporządkowano polski copy w mailach: tematy wiadomości, treść i przypomnienia mają pełne polskie znaki.

7. Pricing behavior across system
- Status: PASS
- Źródło prawdy aktywnej ceny:
  - Supabase `pricing_settings` w live mode,
  - local runtime store tylko w local mode.
- `/admin` zapisuje cenę przez `POST /api/admin/pricing`.
- Landing, admin i formularz pokazują aktywną cenę.
- Nowy booking zapisuje snapshot w `bookings.amount`.
- Checkout i confirmation korzystają z `booking.amount`, więc stary booking zachowuje historyczną kwotę po zmianie ceny.
- `scripts/pricing-smoke.ts` potwierdza:
  - admin price change działa,
  - nowy booking bierze nową cenę,
  - checkout bierze nową cenę,
  - stary booking zachowuje stary snapshot.
