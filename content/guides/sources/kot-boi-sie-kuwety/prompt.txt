## ZADANIA
### 1. Asset PDF
- znajdź aktualny asset / wpis odpowiadający temu produktowi
- podmień lub dodaj nowy asset na podstawie pliku `kot_boi_sie_kuwety_v2.pdf`
- jeśli system wymaga konkretnej lokalizacji public/assets/static, dopasuj go do istniejącej architektury repo
- nie zmieniaj innych PDF-ów

### 2. Dane produktu
- zaktualizuj nazwę, opis, CTA, rolę produktu i cenę zgodnie z `spec_kot_boi_sie_kuwety_v2.md`
- ustaw produkt jako **core PDF**
- jeśli obecny system ma pole typu badge / tag / category / funnel / price / featured, użyj go konsekwentnie

### 3. Karta na stronie
- wdroż copy karty z pliku spec
- CTA główne: `Zobacz PDF`
- CTA pomocnicze: `Sprawdź kolejny krok`
- wyeksponuj wartość: 5 sygnałów, plan 14 dni, pytania do lekarza, checklista, typy kuwet

### 4. Strona produktu / sekcja produktu
- wdroż H1, lead, blok wartości i blok `dla kogo` zgodnie ze spec
- pokaż ten materiał jako główny, uporządkowany PDF ścieżki kuwetowej, nie jako pełny flagship i nie jako szybki darmowy starter
- zachowaj ton: spokojny, premium, klinicznie uporządkowany

### 5. Routing do kolejnych kroków
- dodaj sekcję lub logiczny moduł `Co dalej?` zgodny ze spec
- ma kierować do kolejnych produktów / konsultacji zależnie od dominującego problemu
- nie zmieniaj nazw innych produktów, jeśli nie jest to konieczne w tym kroku

### 6. Spójność lejka
- jeśli obecny układ PDF-ów jest płaski, oznacz ten produkt wyraźnie jako core PDF
- nie rób redesignu całej biblioteki PDF
- uporządkuj tylko tyle, ile trzeba, by ten jeden produkt miał właściwą rolę

### 7. Nie ruszaj
- cen innych PDF-ów
- logiki innych produktów poza minimalnym routingiem do nich
- głównego shellu strony, jeśli jest OK
